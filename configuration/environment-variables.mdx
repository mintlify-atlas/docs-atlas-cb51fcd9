vLLM uses environment variables to configure system behavior, hardware settings, optimization flags, and debugging options.

<Warning>
**Important notes:**

- `VLLM_PORT` and `VLLM_HOST_IP` set the port and IP for vLLM's **internal usage**, not for the API server. Do not use `--host $VLLM_HOST_IP` or `--port $VLLM_PORT` to start the API server.

- All vLLM environment variables are prefixed with `VLLM_`. **Kubernetes users**: Do not name your service `vllm`, as Kubernetes automatically creates environment variables with the capitalized service name as prefix, which will conflict with vLLM's variables.
</Warning>

## Installation time variables

These variables affect vLLM compilation and installation.

### Build configuration

<ParamField path="VLLM_TARGET_DEVICE" type="str" default="cuda">
  Target device for vLLM.
  
  Options: `cuda`, `rocm`, `cpu`
</ParamField>

<ParamField path="VLLM_MAIN_CUDA_VERSION" type="str" default="12.9">
  Main CUDA version for vLLM (follows PyTorch but can be overridden).
</ParamField>

<ParamField path="MAX_JOBS" type="str" default="None">
  Maximum number of parallel compilation jobs.
  
  Defaults to number of CPUs.
</ParamField>

<ParamField path="NVCC_THREADS" type="str" default="None">
  Number of threads for nvcc compilation.
  
  If set, `MAX_JOBS` will be reduced to avoid CPU oversubscription.
</ParamField>

<ParamField path="CMAKE_BUILD_TYPE" type="str" default="None">
  CMake build type.
  
  Options: `Debug`, `Release`, `RelWithDebInfo`
</ParamField>

<ParamField path="VERBOSE" type="bool" default="false">
  Print verbose logs during installation.
</ParamField>

### Precompiled binaries

<ParamField path="VLLM_USE_PRECOMPILED" type="bool" default="false">
  Use precompiled binaries (*.so files).
</ParamField>

<ParamField path="VLLM_SKIP_PRECOMPILED_VERSION_SUFFIX" type="bool" default="false">
  Skip adding +precompiled suffix to version string.
</ParamField>

## Runtime variables

These variables configure vLLM's runtime behavior.

### Cache and storage

<ParamField path="VLLM_CACHE_ROOT" type="str" default="~/.cache/vllm">
  Root directory for vLLM cache files.
  
  Respects `XDG_CACHE_HOME` if set.
</ParamField>

<ParamField path="VLLM_CONFIG_ROOT" type="str" default="~/.config/vllm">
  Root directory for vLLM configuration files.
  
  Respects `XDG_CONFIG_HOME` if set.
</ParamField>

<ParamField path="VLLM_ASSETS_CACHE" type="str" default="~/.cache/vllm/assets">
  Path to cache for storing downloaded assets.
</ParamField>

### Distributed execution

<ParamField path="VLLM_HOST_IP" type="str" default="">
  IP address of the current node for distributed execution.
  
  Set this differently on each node when using multi-node inference.
</ParamField>

<ParamField path="VLLM_PORT" type="int" default="None">
  Port for vLLM internal communication.
  
  <Warning>
  This is NOT the API server port. This is for internal distributed communication only.
  </Warning>
</ParamField>

<ParamField path="VLLM_NCCL_SO_PATH" type="str" default="None">
  Path to NCCL library file.
  
  Needed because nccl>=2.19 from PyTorch may contain bugs.
</ParamField>

### Logging

<ParamField path="VLLM_CONFIGURE_LOGGING" type="bool" default="true">
  Whether vLLM should configure logging.
  
  Set to `0` to disable vLLM's logging configuration.
</ParamField>

<ParamField path="VLLM_LOGGING_LEVEL" type="str" default="INFO">
  Default logging level.
  
  Options: `DEBUG`, `INFO`, `WARNING`, `ERROR`, `CRITICAL`
</ParamField>

<ParamField path="VLLM_LOGGING_CONFIG_PATH" type="str" default="None">
  Path to logging config JSON file for both vLLM and uvicorn.
</ParamField>

<ParamField path="VLLM_LOGGING_COLOR" type="str" default="auto">
  Control colored logging output.
  
  Options: `auto`, `1` (always), `0` (never)
</ParamField>

<ParamField path="NO_COLOR" type="bool" default="false">
  Standard Unix flag for disabling ANSI color codes.
</ParamField>

<ParamField path="VLLM_LOG_STATS_INTERVAL" type="float" default="10.0">
  Interval in seconds to log statistics.
</ParamField>

### Model loading

<ParamField path="VLLM_USE_MODELSCOPE" type="bool" default="false">
  Load models from ModelScope instead of Hugging Face Hub.
</ParamField>

<ParamField path="VLLM_MODEL_REDIRECT_PATH" type="str" default="None">
  Path to JSON file or space-separated values table mapping model repo IDs to local folders.
  
  Example JSON:
  ```json
  {"meta-llama/Llama-3.2-1B": "/tmp/Llama-3.2-1B"}
  ```
</ParamField>

<ParamField path="HF_TOKEN" type="str" default="None">
  Hugging Face API token for private models.
</ParamField>

### Engine configuration

<ParamField path="VLLM_ENGINE_ITERATION_TIMEOUT_S" type="int" default="60">
  Timeout for each iteration in the engine (seconds).
</ParamField>

<ParamField path="VLLM_ENGINE_READY_TIMEOUT_S" type="int" default="600">
  Timeout for engine cores to become ready during startup (seconds).
</ParamField>

<ParamField path="VLLM_ALLOW_LONG_MAX_MODEL_LEN" type="bool" default="false">
  Allow max sequence length greater than the max length from model config.
</ParamField>

### Multi-modal settings

<ParamField path="VLLM_IMAGE_FETCH_TIMEOUT" type="int" default="5">
  Timeout for fetching images when serving multi-modal models (seconds).
</ParamField>

<ParamField path="VLLM_VIDEO_FETCH_TIMEOUT" type="int" default="30">
  Timeout for fetching videos (seconds).
</ParamField>

<ParamField path="VLLM_AUDIO_FETCH_TIMEOUT" type="int" default="10">
  Timeout for fetching audio (seconds).
</ParamField>

<ParamField path="VLLM_MAX_AUDIO_CLIP_FILESIZE_MB" type="int" default="25">
  Maximum audio file size in MB for speech-to-text requests.
</ParamField>

### GPU and memory

<ParamField path="CUDA_VISIBLE_DEVICES" type="str" default="None">
  Control visible GPU devices.
</ParamField>

<ParamField path="VLLM_FLOAT32_MATMUL_PRECISION" type="str" default="highest">
  PyTorch float32 matmul precision mode.
  
  Options: `highest`, `high`, `medium`
</ParamField>

<ParamField path="VLLM_FUSED_MOE_CHUNK_SIZE" type="int" default="16384">
  Chunk size for fused MoE operations.
</ParamField>

### Compilation and optimization

<ParamField path="VLLM_USE_AOT_COMPILE" type="bool" default="auto">
  Enable ahead-of-time compilation.
  
  Automatically enabled on PyTorch >= 2.10.0.
</ParamField>

<ParamField path="VLLM_DISABLE_COMPILE_CACHE" type="bool" default="false">
  Disable torch.compile cache.
</ParamField>

<ParamField path="VLLM_ENABLE_INDUCTOR_MAX_AUTOTUNE" type="bool" default="true">
  Enable Inductor max_autotune for better performance.
</ParamField>

### Debugging

<ParamField path="VLLM_TRACE_FUNCTION" type="int" default="0">
  Trace function calls for debugging.
  
  Set to `1` to enable.
</ParamField>

<ParamField path="VLLM_DEBUG_DUMP_PATH" type="str" default="None">
  Dump FX graphs to the specified directory for debugging.
</ParamField>

<ParamField path="VLLM_PATTERN_MATCH_DEBUG" type="str" default="None">
  Debug pattern matching inside custom passes.
  
  Set to fx.Node name (e.g., 'getitem_34').
</ParamField>

### Platform-specific

#### CPU backend

<ParamField path="VLLM_CPU_KVCACHE_SPACE" type="int" default="0">
  CPU key-value cache space in GB.
  
  Default is 4 GB if not set.
</ParamField>

<ParamField path="VLLM_CPU_OMP_THREADS_BIND" type="str" default="auto">
  CPU core IDs bound by OpenMP threads.
  
  Examples: `"0-31"`, `"0,1,2"`, `"0-31|32-63"`
</ParamField>

#### ROCm backend

<ParamField path="VLLM_ROCM_SLEEP_MEM_CHUNK_SIZE" type="int" default="256">
  Chunk size in MB for sleeping memory allocations under ROCm.
</ParamField>

<ParamField path="VLLM_ROCM_FP8_PADDING" type="bool" default="true">
  Pad FP8 weights to 256 bytes for ROCm.
</ParamField>

<ParamField path="VLLM_ROCM_CUSTOM_PAGED_ATTN" type="bool" default="true">
  Use custom paged attention kernel for MI3* cards.
</ParamField>

#### XLA/TPU

<ParamField path="VLLM_XLA_CACHE_PATH" type="str" default="~/.cache/vllm/xla_cache">
  Path to XLA persistent cache directory.
</ParamField>

<ParamField path="VLLM_XLA_USE_SPMD" type="bool" default="false">
  Enable SPMD mode for TPU backend.
</ParamField>

### Ray settings

<ParamField path="VLLM_USE_RAY_COMPILED_DAG_CHANNEL_TYPE" type="str" default="auto">
  Channel type for Ray Compiled Graph communication.
  
  Options: `auto`, `nccl`, `shm`
</ParamField>

<ParamField path="VLLM_WORKER_MULTIPROC_METHOD" type="str" default="fork">
  Multiprocess context for workers.
  
  Options: `fork`, `spawn`
</ParamField>

### Usage statistics

<ParamField path="VLLM_NO_USAGE_STATS" type="bool" default="false">
  Disable usage statistics collection.
</ParamField>

<ParamField path="VLLM_DO_NOT_TRACK" type="bool" default="false">
  Alternative flag to disable usage tracking.
</ParamField>

<ParamField path="VLLM_USAGE_STATS_SERVER" type="str" default="https://stats.vllm.ai">
  Server URL for usage statistics.
</ParamField>

## Usage examples

### Set logging level

```bash
export VLLM_LOGGING_LEVEL=DEBUG
vllm serve meta-llama/Llama-3.1-8B-Instruct
```

### Configure multi-node setup

```bash
# On node 1
export VLLM_HOST_IP=192.168.1.10
vllm serve MODEL --tensor-parallel-size 4

# On node 2
export VLLM_HOST_IP=192.168.1.11
vllm serve MODEL --tensor-parallel-size 4
```

### Enable debugging

```bash
export VLLM_TRACE_FUNCTION=1
export VLLM_DEBUG_DUMP_PATH=/tmp/vllm_debug
vllm serve MODEL
```

### Use ModelScope instead of Hugging Face

```bash
export VLLM_USE_MODELSCOPE=true
vllm serve Qwen/Qwen2.5-7B-Instruct
```

### Disable usage statistics

```bash
export VLLM_NO_USAGE_STATS=1
vllm serve MODEL
```
